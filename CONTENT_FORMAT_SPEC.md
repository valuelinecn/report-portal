## 一、研究报告模板 (viz_full.html)

### 1.1 文件命名
```
{company_pinyin}_viz_full.html
```
- 公司名用拼音全拼（如：`maotai`、`zhongjixuchuang`、`wuliangye`）
- 文件名必须带 `_viz_full` 后缀

### 1.2 HTML 结构规范

**调色板（色系规则）**
| 行业类型 | 主题色 | 使用位置 |
|---------|-------|---------|
| 消费/传统/价值股 | 金色 `#f0c040` | h2标题、评分环、高亮文字 |
| 科技/成长/AI | 青色 `#4fc3f7` | h2标题、评分环、高亮文字 |
| 金融/银行 | 蓝色 `#2196f3` | (如适用) |
| 医药 | 绿色 `#4caf50` | (如适用) |

**背景色**：`#0a0a1a`（深色底），卡片色 `#12122a` / `#0d1a2a`（根据色系微调）
**边框**：`#1e1e3a`（消费）/ `#1a2a3a`（科技）

### 1.3 导航条（sticky-top）
必须包含以下固定锚点（ID 从 `#s0` 到 `#s13`）：
```
评分 | 综合评价 | M1商业模式 | M2壁垒 | M3管理层
M4财务 | M5产业链 | M6估值 | M7市占率
M8催化剂 | M9预期差 | M10风险 | 综合评级 | 附录
```

### 1.5 CSS 类名规范（必须使用）
| CSS类 | 用途 |
|-------|------|
| `.sc-wrap` | 评分环容器 |
| `.score-ring` | 评分环（conic-gradient） |
| `.score-inner` + `.score-num` + `.score-label` | 环内数字 |
| `.card` | 模块卡片容器 |
| `.tbl` | 数据表格（th用主题色） |
| `.mg` + `.mi` + `.mv` + `.ml` | 指标网格 |
| `.hb` + `.hl` + `.ht` + `.hf` + `.hv` | 横向条形图（**重要：长标签放.hv，短标签放.hf**） |
| `.scn` + `.scc` + `.sc1/.sc2/.sc3` | 三情景对比卡片 |
| `.cp` + `.nm` + `.bt` + `.bf` + `.vl` | 对比条形图 |
| `.bq` | 引用/结论块（带主题色左边框） |
| `.txt` + `.dot` | 文本段落 |
| `.tg` + `.tgg/.tgr/.tgo/.tgy` | 标签徽章 |
| `.gold` / `.green` / `.red` / `.orange` | 文字颜色 |
| `.dim-row` + `.dim-label` + `.dim-bg` + `.dim-fill` + `.dim-sc` | 评分维度条 |

### 1.6 横向条形图文字防溢出规则（重要）
```html
<!-- ✅ 正确：短标签（≤5字符）放 .hf 内 -->
<div class="hb">
  <span class="hl">品牌</span>
  <div class="ht"><div class="hf" style="width:90%;background:#4fc3f7;">4.5</div></div>
</div>

<!-- ✅ 正确：长描述放 .hv 外 -->
<div class="hb">
  <span class="hl">系列酒</span>
  <div class="ht"><div class="hf" style="width:13%;background:linear-gradient(90deg,#4caf50,#2e7d32);"></div></div>
  <span class="hv">13% · 约250亿</span>
</div>
```
**规则**：宽度 ≤ 15% 的条，标签文字一律放 `.hv`（外置）；宽度 ≥ 40% 的文字可以放 `.hf`（内置）。

### 1.7 数据表格必须包含
- 年度财报表：**6年**（2020-最新完整年），列：年份/营收/YoY/净利/净利率/毛利率
- 最新年份行用 `.gold` 高亮
- 增速正值用 `.green`、负值用 `.red`

### 1.8 文件大小要求
- 完整版 viz HTML 必须 **≥ 45KB**（含全部 M1-M11 文字解读 + 图表）
- 低于 40KB 视为简化版，需要重新生成

## 三、书籍板块模板 (books-data.js)

### 3.1 数据结构
```javascript
{
  id: 'kebab-case-id',           // 唯一标识
  title: '中文书名',
  titleEn: 'English Title',
  author: '作者名',
  authorEn: 'Author English',
  year: 1949,                    // 出版年份
  rating: 5,                     // 1-5 星
  coverEmoji: '📘',              // 单个 emoji
  tags: ['标签1', '标签2'],      // 2-3 个
  color: '#f0c040',              // 主题色
  intro: '2-3段简介...',
  architecture: [
    {
      title: '第X部分：标题',
      chapters: [
        {
          title: '第X章 标题',
          content: '2-4段详细内容...'
        }
      ],
      summary: '部分总结...'
    }
  ],
  quotes: [
    { text: '金句内容...', context: '来源（如 第8章）' }
  ],
  summary: '全书总结...'
}
```

### 3.2 章节内容要求
- 每章内容 **2-4段**，每段 100-300 字
- 内容必须反映书籍**实际内容**（概念、案例、建议），不能泛泛而谈
- 每个部分的章节数：**3-7章**
- 全书总章数：**10-20章**
- 金句数量：**8-10条**，带来源标注
- 金句不加引号（直接作为 JS 字符串即可）

### 3.3 命名规范
- `id`: 英文小写-kebab-case（如 `poor-charlie`、`intelligent-investor`）
- 不要用特殊字符
- 书名用中文原名（完整名称）

### 3.4 质量要求
- 写入后必须验证：`node -c books-data.js`
- 每本书先确认已有资料完整性，不足时先搜索补充再写

## 四、门户索引规范 (index.html)

### 4.1 COMPANIES 数组条目格式
```javascript
{
  id: 'maotai',                   // 拼音，匹配 reports/{id}.html
  name: '贵州茅台',
  nameEn: 'Kweichow Moutai',
  score: 7.0,                     // 评分 /10
  scoreMax: 10,
  color: '#f0c040',              // 消费金/科技青
  industry: '白酒 · 高端消费',
  desc: '中国高端白酒行业绝对龙头',
  tags: ['品牌龙头', '高毛利率', '核心资产'],
  metrics: {营收: '1,899亿', 净利: '910亿', 毛利率: '91.7%', PE: '20.7x', ROE: '28.5%'},
  file: 'maotai.html',
  type: 'full'
}
```

### 4.2 色系规则
| 行业 | color值 |
|------|--------|
| 消费/白酒/传统 | `#f0c040`（金色） |
| 科技/AI/通信 | `#4fc3f7`（青色） |
| 金融/银行 | `#2196f3`（蓝色） |
| 医药/生物 | `#4caf50`（绿色） |
| 新能源/制造 | `#ff9800`（橙色） |

### 4.3 metrics 字段必须包含
每个条目必须包含：`营收`、`净利`、`毛利率`、`PE`、`ROE`（5个核心指标）
数据来自报告中的最新完整年度（2025年）

### 4.4 添加新企业流程
1. 复制 viz_full.html → `reports/{pinyin}.html`
2. 向 `COMPANIES` 数组插入新条目
3. `git add -A && git commit -m "feat: add {公司名} report" && git push`

## 六、"更换企业" 完整工作流

当用户说 "更换企业 {公司名}" 时，必须按此顺序执行：

```
1. delegate_task 搜索最新数据（分3轮）
   └─ 第1轮：核心财务+行业数据
   └─ 第2轮：补充数据
   └─ 第3轮（数据冲突时）：API核验
2. 数据校对（处理冲突，取保守值）
3. 生成 {company}_viz_full.html（≥45KB，含全部M1-M11）
4. 生成 xiaohongshu_{company}.md
5. 复制到 /storage/emulated/0/Download/企业分析/
6. 更新报告门户
   └─ 复制HTML到 reports/{pinyin}.html
   └─ COMPANIES 数组添加条目
   └─ git commit + push（如果已配置GitHub Pages）
7. 报告进度（每完成一步告知用户）
```
