# 报告生成流程 v40

## 一、准备阶段
```
1. 复制 blank-template.html → reports/{股票代码}.html
2. 检查年报/季报PDF在工作目录
3. 检查研报客技能、fin-table API 可用
```

## 二、数据采集（按优先级）
```
4. 年报PDF提取（财务/管理层/分业务/行业分析/竞争格局/风险）
5. 季报PDF提取（最新季度数据）
6. fin-table API（历史多年度结构数据）
7. 研报客 search --type content（补充年报未覆盖内容）
8. 仍缺失 → 一次性汇总问用户
```

## 三、填充阶段
```
9. 预先诊断：列出模板所有 h3 标题和表格状态
   grep -oP '<h3[^>]*>[^<]+</h3>' reports/{文件}
   python3 -c "检查每个表格是否有数据行、空单元格"

10. 按 h3 标题逐个填充：
    - 基础信息 → html.replace('{{KEY}}', 'value')
    - 表格 → replace_table() 整块替换（从 <h3>标题</h3> 到 </table>）
    - 没有 h3 标题的表格 → 找到上下文的唯一标识，整体替换
    - 不确定的内容 → 问用户，不猜

11. 确认无未填 h3 或空表格：
    - 重复第9步的诊断命令，确认每个 h3 对应的表格都有数据行
    - 空单元格 <td></td> = 0

12. BQ 12个按 M1-M10 顺序逐个填入唯一结论

13. 清理剩余 {{...}} 占位符
```

## 四、验证清单
```
[ ] 第9步：h3标题已列出
[ ] 第10步：每个h3对应表格已替换
[ ] 第11步：无未填h3或空表格
[ ] 空单元格 <td></td> = 0
[ ] 占位符 {{...}} = 0
[ ] div平衡（opens == closes）
[ ] BQ全部唯一（12个无重复）
[ ] 无旧公司残留
[ ] JS语法通过（node --check）
[ ] 所有 <table> 含 class="tbl"
[ ] 四张CARD卡（综合评级/风险等级/成长弹性/ROE）
```

## 五、预览与推送
```
14. 本地 http.server 启动，用户确认渲染正常
15. 推 test 分支
16. 用户确认后 deploy-to-main.sh
17. 输出来源清单（每个数据标注出处）
```
